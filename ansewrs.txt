Exercise 2: Starting Processes in the Background
------------------------------------------------
Commands:
- sleep 300 &   --> Starts a process in the background for 300 seconds.
- jobs          --> Lists all running background processes.

Explanation:
- The & operator allows the process to run in the background.
- jobs command is used to monitor background processes.

---

Exercise 3: Stopping Processes
------------------------------
Commands:
- ps aux | grep sleep  --> Find the PID of the sleep process.
- kill <PID>           --> Stop/terminate the process.
- ps aux | grep sleep  --> Verify the process is no longer running.

Explanation:
- ps lists all processes.
- kill sends a signal to terminate the process.

---

Exercise 4: Pausing and Resuming a Process
-----------------------------------------
Commands:
- kill -STOP <PID>  --> Pause the process.
- kill -CONT <PID>  --> Resume the process.

Explanation:
- STOP signal suspends the process temporarily.
- CONT signal resumes the suspended process.

Exercise 5: Role of the Linker
------------------------------
Files: file1.c & file2.c

Description:
- file1.c contains function hello() that prints a message.
- file2.c calls hello() in main().
- Compile and link: gcc file1.c file2.c -o output_program

Explanation:
- The linker combines multiple object files into a single executable.
- Resolves references between functions and variables across files.
- Modifying one file and recompiling only that file saves time during linking.

Job of the Linker:
- Combines object files (.o) into one executable.
- Resolves external references to functions and variables.

---

Exercise 6: Role of the Loader
------------------------------
File: simple_program.c

Description:
- Prints a simple message: "This is a simple program."
- Compile: gcc simple_program.c -o simple_program
- Check dynamic libraries: ldd simple_program

Explanation:
- The loader loads the program into memory when executed.
- It links the executable with dynamic libraries at runtime.

Job of the Loader:
- Loads the executable into memory.
- Prepares it for execution by linking with required libraries.
